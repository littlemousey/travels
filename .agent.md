# Reisherinneringen Project Agent Instructions

This repository contains an interactive travel memories application showcasing Ans' travels from 1991. This is a React + TypeScript application.

## Project Overview

**Purpose**: Display travel memories on an interactive map with scrollable stories
**Tech Stack**: React 18, TypeScript, Vite, Emotion (styled-components), MapLibre GL
**State Management**: React Context API
**Styling**: Emotion (CSS-in-JS) with a vintage travel journal aesthetic

## Architecture Principles

### Separation of Concerns
- **Data**: All travel data lives in `src/data/` (chapters.ts, markers.ts)
- **Types**: TypeScript definitions in `src/types/index.ts`
- **Context**: Global state in `src/context/ChapterContext.tsx`
- **Components**: UI components organized by domain (Map, Story, Layout)
- **Styles**: Global styles and theme in `src/styles/GlobalStyles.tsx`
- **Hooks**: Reusable logic in `src/hooks/`

### Component Structure
```
components/
├── Map/
│   ├── Map.tsx              # Main map component with MapLibre GL
│   └── MapOverlay.tsx       # Title overlay and vignette effect
├── Story/
│   ├── Story.tsx            # Story container with scroll handling
│   ├── StoryHeader.tsx      # Header section
│   ├── Chapter.tsx          # Individual chapter cards
│   └── StoryFooter.tsx      # Footer section
└── Layout/
    └── TravelMemories.tsx   # Main layout combining Map + Story
```

## Key Patterns

### Adding New Travel Chapters

1. **Add chapter data** in `src/data/chapters.ts`:
   ```typescript
   {
     id: 'unique-id',
     title: 'Location Name',
     date: 'Month Year',
     center: [longitude, latitude],
     zoom: 10,
     pitch: 40,
     bearing: 0,
     year: '1994',
     period: 'Summer',
     flag: '🇪🇸',
     heading: 'Chapter Title',
     locationTag: 'City, Country',
     content: 'Story text...',
     stops: ['Stop 1', 'Stop 2'], // Optional
     markerIndices: [9, 10], // Array of marker indices
   }
   ```

2. **Add markers** in `src/data/markers.ts`:
   ```typescript
   { coords: [lng, lat], label: 'Place Name', sub: 'Country' }
   ```

3. The UI will automatically update via Context API

### State Flow

1. User scrolls in Story component
2. `useScrollChapter` hook detects which chapter is in view
3. Hook calls `setCurrentChapterIndex` from Context
4. Map component reacts to context change and flies to new location
5. Markers update their active state based on chapter's `markerIndices`

### Styling with Emotion

- Use styled components for all styling
- Access theme via `import { theme } from '../../styles/GlobalStyles'`
- Keep component-specific styles co-located with components
- Global styles (like MapLibre popup styles) go in GlobalStyles.tsx

## Common Tasks

### Modifying Map Behavior
Edit `src/components/Map/Map.tsx`:
- Map initialization in first `useEffect`
- Marker creation in map 'load' event
- Camera animations in second `useEffect` (when chapter changes)

### Updating Story Content
Edit `src/data/chapters.ts` - no component changes needed

### Changing Visual Design
- Theme colors: `src/styles/GlobalStyles.tsx`
- Component styles: Each component's styled-components
- Typography: Update font imports in `index.html`

### Debugging Scroll Behavior
Check `src/hooks/useScrollChapter.ts`:
- Ensure `data-chapter-index` attribute is on chapter elements
- Adjust visibility calculation if scroll detection is off
- Check header offset calculation

## Known Implementation Details

### MapLibre Integration
- Map instance stored in `mapRef`
- Markers stored in `markersRef` array
- Cleanup handled in `useEffect` return
- Custom marker styling applied via DOM manipulation

### Scroll Detection
- Uses intersection-based visibility calculation
- Header section treated specially (always shows chapter 0)
- Most visible chapter becomes active

### TypeScript Configuration
- `jsxImportSource: "@emotion/react"` for Emotion
- Strict mode enabled
- All components fully typed

## File Naming Conventions
- React components: PascalCase (e.g., `StoryHeader.tsx`)
- Data files: camelCase (e.g., `chapters.ts`)
- Hooks: camelCase with 'use' prefix (e.g., `useScrollChapter.ts`)
- Types: camelCase interfaces in PascalCase (e.g., `Chapter` interface)

## Deployment

### GitHub Pages
The project is configured for GitHub Pages deployment:

```bash
npm run deploy  # Build and deploy to gh-pages branch
```

**Configuration:**
- `vite.config.ts`: `base: '/travels/'` - sets the base path for GitHub Pages
- `package.json`: deploy script uses `gh-pages` package
- Output: `dist/` folder deployed to `gh-pages` branch
- Live URL: `https://[username].github.io/travels/`

**First-time setup:**
1. Repository must be named `travels` (or update `base` in vite.config.ts)
2. Enable GitHub Pages in repo settings (Source: gh-pages branch)
3. Run `npm run deploy`

**Build Process:**
1. TypeScript compilation (`tsc`)
2. Vite production build
3. `gh-pages` pushes `dist/` to gh-pages branch

## Important Notes
- Map animations use duration: 2800ms, curve: 1.4 for smooth flyTo
- Mobile responsive: switches to vertical layout via @media query
- All travel data is stored in `src/data/` for easy updates
